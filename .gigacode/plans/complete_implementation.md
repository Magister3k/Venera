### 1.2 Реализовать полную бизнес-логику в processes/manager.go

**Цель:** Реализовать управление процессами с таймерами и каналами (п.4 ТЗ)

**Бизнес-логика (п.4 ТЗ):**
- Процессы ПВ1...ПВn получают данные, заносят в list ЛСТ1...ЛСТn DragonflyDB
- При достижении порога ПЗ или срабатывании таймера ТП1...ТПn отправляется сигнал в канал КАН1...КАНn
- Процесс управления задачами запускает процессы обработки ПО1...ПОn для фильтрации и переноса данных
- Процессы обработки: получение из list -> разделение -> фильтрация -> sorted set -> PostgreSQL

**Задачи:**
- Добавить таймеры ТП1...ТПn для периодической обработки
- Реализовать каналы КАН1...КАНn для синхронизации
- Создать процессы обработки ПО1...ПОn для фильтрации данных
- Интеграция с DragonflyDB (list/sorted set)
- Интеграция с PostgreSQL (pgx v5)

**Ожидаемые файлы:**
- `processes/manager.go` — полная реализация менеджера процессов

**Критерии завершения:**
- Запуск/остановка процессов
- Работа таймеров и каналов
- Передача данных между модулями
- Обработка ошибок и edge cases
