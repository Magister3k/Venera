## Критические проблемы в текущем коде

### 1. main.go
- **Проблема**: Используется `unsafe` пакет без необходимости
- **Проблема**: Неполная реализация `hasRunningProcess()` - мьютекс не удаляется при выходе
- **Проблема**: Отсутствует обработка `config.LoadControlFile()` и `config.LoadAlertsFile()`
- **Проблема**: Метод `ensurePodmanRunning()` использует неправильный подход для проверки

### 2. processes/manager.go
- **Проблема**: Отсутствует реализация бизнес-логики процессов (п.4 ТЗ)
- **Проблема**: Неполная реализация `ProcessQueue` и `DataProcessor`
- **Проблема**: Отсутствует механизм сигналов между процессами
- **Проблема**: Неполная реализация `Tshark` и `DataSelector`

### 3. data/parser.go
- **Проблема**: Используется `msgpack` вместо `fastjson` как указано в ТЗ
- **Проблема**: `ParseJSON()` возвращает пустой массив
- **Проблема**: `FormatData()` неправильно преобразует timestamp в строку

### 4. data/filter.go
- **Проблема**: Черный список не загружается из файла (загружается в белый список)
- **Проблема**: Метод `LoadFromFile()` не выделяет черный список правильно

### 5. services/win_service.go
- **Проблема**: Отсутствует реализация `RunService()` для Windows
- **Проблема**: `hStatusHandle` объявлен как глобальная переменная, но не инициализирован
- **Проблема**: Метод `setServiceStatus()` ссылается на несуществующую переменную

### 6. web/server.go
- **Проблема**: Множество методов помечены как TODO
- **Проблема**: Отсутствует WebSocket реализация
- **Проблема**: Неполная реализация API endpoints

### 7. notify/alerts.go
- **Проблема**: Методы `CheckCondition()` и `checkConditionSimple()` возвращают false
- **Проблема**: Отсутствует реализация проверки условий CEL
